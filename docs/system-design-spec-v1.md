## 1. 系统背景与设计思路

### 1.1 社区卫生服务中心的业务特点

社区卫生服务中心不同于三甲医院，其核心工作是：

- **日常门诊**：处理头疼脑热、慢性病复诊等基础医疗需求，门诊量有限
- **公共卫生管理**：为辖区内居民（尤其是高血压、糖尿病等慢性病患者、老年人、孕产妇、儿童）建立健康档案并定期随访
- **家庭医生签约**：居民与特定医生建立长期签约关系，由该医生负责其健康管理
- **疫苗接种**：承担辖区计划免疫工作，主要面向儿童和需要特定疫苗的成人

因此，系统设计的核心不是挂号取号的流量管理，而是**长期健康管理与跟踪**。

### 1.2 功能设计原则

- 以**诊疗闭环**为主线：预约 → 接诊 → 开具处方与随访计划 → 居民查看结果与随访提醒 → 闭环
- 三端分工明确：居民端面向患者，医护端面向医生和护士日常工作，管理员端面向运营配置
- 医生与护士共用医护端入口，通过角色权限控制各自可见和可操作的功能
- 功能精简，不追求数量，每个模块有清晰的业务存在价值

## 2. 角色定义与权限模型

### 2.1 系统角色一览

| 角色标识 | 角色名称 | 入口 | 说明 |
|---|---|---|---|
| ROLE_RESIDENT | 居民 | 居民端 | 在社区卫生中心辖区内注册的普通居民，使用系统进行预约、查看记录等操作 |
| ROLE_DOCTOR | 医生 | 医护端 | 负责接诊、诊断、开具处方、制定随访计划 |
| ROLE_NURSE | 护士 | 医护端 | 负责叫号登记、疫苗接种记录、执行随访联系任务 |
| ROLE_ADMIN | 管理员 | 管理员端 | 负责系统配置、用户管理、排班管理、数据查看 |

### 2.2 医护端权限分级说明

医生和护士共用同一个医护端入口，登录时由系统识别角色，动态渲染各自可见的导航菜单和操作按钮。权限控制在后端接口层面实现，前端根据角色字段控制菜单和按钮的显示与隐藏。

| 功能操作 | 医生 | 护士 |
|---|---|---|
| 接诊（填写诊断 / 开处方） | ✅ 可操作 | ❌ 无权限 |
| 协助叫号 / 登记到诊 | 可查看 | ✅ 可操作 |
| 查看居民完整健康档案 | ✅ 可操作 | 仅基础信息 |
| 录入慢病指标数据 | ✅ 可操作 | ❌ 无权限 |
| 制定随访计划 | ✅ 可操作 | ❌ 无权限 |
| 执行随访记录（电话 / 上门） | 可查看 | ✅ 可操作 |
| 接种管理（记录接种） | ❌ 无权限 | ✅ 可操作 |
| 查看处方记录 | ✅ 可操作 | 只读查看 |
| 查看本人排班 | ✅ 可查看 | ✅ 可查看 |

## 3. 居民端功能模块

居民端共 **7 个功能模块**（含首页）。首页作为聚合入口，不单独计入核心功能模块数量。

### 3.1 首页

展示系统快捷入口（预约、查档案等）；展示待办提醒（即将到来的随访日期、疫苗接种到期提醒）；展示系统公告。

### 3.2 预约挂号

按科室浏览医生列表；选择时间段完成预约；已签约的家庭医生优先展示在列表顶部；支持取消预约；支持为家庭成员代预约（需已添加成员并完成授权）。

### 3.3 就诊记录

查看历史就诊详情，包含就诊日期、科室、医生、诊断内容、处方；查看医生为该次就诊生成的随访计划及提醒通知。随访提醒合并展示在此处，不单独设立独立的随访模块。

> 关联创新点①②

### 3.4 我的档案

查看个人基础健康信息（身高、体重、血型、过敏史等）；查看慢病健康指标的历史趋势图（折线图展示血压、血糖等指标的时间变化）；查看当前家庭医生签约状态，包含签约医生姓名和有效期；支持发起签约申请或解约申请。

> 关联创新点①

### 3.5 疫苗接种

查看本人全部接种历史记录（疫苗名称、接种日期、接种医护）；查看下次接种提醒；跳转预约接种入口。

### 3.6 家庭成员

添加家庭成员（填写姓名、身份证、关系）；为成员申请代管授权，授权范围包括代为预约和代查就诊记录；设置敏感信息隔离，特定科室的就诊记录对代管人不可见；被代管人可随时撤销授权。

> 关联创新点③

### 3.7 个人中心

维护本人姓名、手机号、身份证、头像等基本信息；修改登录密码；查看登录设备记录。

## 4. 医护端功能模块

医护端共 **7 个功能模块**（含个人中心）。医生和护士共用同一入口，角色不同则可见模块和操作权限不同，详见第 2 节。

### 4.1 接诊工作台

**医生视角：** 查看当日预约列表；呼叫下一位患者（叫号）；填写本次诊断内容；开具处方（药品名称、剂量、用法）；接诊完成后系统自动生成就诊记录。

**护士视角：** 查看当日预约列表；协助登记患者到诊状态；叫号辅助操作；不可填写诊断和处方。

### 4.2 居民档案

**医生视角：** 查看居民完整健康档案，包含基础信息、历史就诊、慢病指标数据；在档案中录入本次就诊的慢病指标（血压、血糖、体重等）。

**护士视角：** 查看居民基础健康信息（姓名、年龄、过敏史等）；不可查看完整诊断历史；不可录入指标数据。

### 4.3 签约与随访

> 即原需求中的"随访公卫"模块，改名为"签约与随访"更直观。含义是：医生管理自己的签约居民，并针对慢病患者开展定期随访工作。两者在业务上高度关联，合为一个模块。

**医生视角：** 查看本人名下所有签约居民列表；针对慢病患者生成随访计划（系统根据病种提供模板，医生确认后推送给居民）；查看随访异常预警（指标连续异常的居民会被标记）；查看待随访任务列表。

**护士视角：** 查看待随访任务列表；执行随访任务，记录电话联系或上门随访的过程和居民反馈；填写随访结果后推进任务状态至已完成。

### 4.4 接种管理

**护士专属模块，医生无此入口。**

查看当日预约接种列表；核对居民接种记录与禁忌；执行接种并录入接种结果（疫苗批次、接种部位、执行人）；查看接种计划和历史记录。

### 4.5 处方记录

**医生视角：** 查看本人历史全部开方记录，支持按居民、日期筛选。

**护士视角：** 只读查看处方记录，不可开具或修改处方。

### 4.6 我的排班

查看本人当周及下周排班安排（科室、时间段）；提交调班申请。医生和护士权限相同。

### 4.7 个人中心

维护个人基本信息；修改密码。医生和护士权限相同。

## 5. 管理员端功能模块

管理员端共 **6 个功能模块**。管理员不参与具体诊疗业务，职责是系统运营配置和全局数据查看。

### 5.1 数据看板（首页）

展示今日门诊量、预约总数、各科室当日预约饱和度（已预约数 / 总号源数）；展示随访完成率和待随访任务数；展示本月签约新增数量。数据以卡片和简单图表方式呈现，作为管理员登录后的默认视图。

### 5.2 用户管理

创建医护账号（填写姓名、手机号、所属科室、角色）；停用或启用账号；重置账号密码；查看账号列表；分配角色（医生或护士）。

### 5.3 排班管理

为医生和护士配置排班（科室 + 日期 + 时间段）；设置每个时段的最大接诊号源数量；查看排班冲突提醒（同一人在同一时段被重复排班时系统标记）；处理医护提交的调班申请（批准或驳回）。

### 5.4 科室管理

维护科室基础信息：科室名称、科室编号、科室简介与职能说明、所属楼层与位置、科室负责人（关联已有医生账号）；设置科室状态（正常接诊 / 暂停接诊）。科室停诊时，居民端预约挂号中该科室自动不可选。

### 5.5 居民管理

查看系统内注册居民列表（姓名、手机号、身份证号、注册时间）；查看居民的家庭代管关系；处理异常账号（停用）。不包含编辑居民健康档案，健康数据由医护负责录入和维护。

### 5.6 签约管理

查看全系统所有签约记录（居民与医生的对应关系、签约时间、有效期）；审核居民发起的签约申请（批准或驳回）；统计各医生当前签约居民数量，供管理员评估各医生的负载情况。

## 6. 三大创新点实现说明

### 6.1 创新点① 慢病健康趋势追踪

**背景与价值**

高血压、糖尿病是社区卫生中心服务的核心慢病人群，需要长期监测血压、血糖、体重等指标。传统系统只存储单次就诊数据，无法直观展示健康变化趋势，医生难以快速判断患者病情走势。

**实现方向**

- **数据录入**：医生每次接诊或随访时，在居民档案模块录入本次的慢病指标，字段包含血压（收缩压 / 舒张压）、空腹血糖、体重，每条记录带有录入时间戳
- **数据存储**：为慢病指标建立独立的数据表，记录指标类型、数值、时间、录入医生等信息，与就诊记录关联
- **趋势展示**：居民端我的档案模块使用折线图展示最近若干次指标数据的变化曲线，横轴为时间，纵轴为指标值，前端使用图表库（如 ECharts）实现
- **异常预警**：在指标录入时触发检测逻辑，若某指标连续多次超出正常阈值范围（如收缩压连续 3 次高于 140mmHg），则在签约与随访模块中对该居民标记异常，医生在随访时优先看到

**涉及模块**

- 居民端：我的档案（展示趋势图）
- 医护端：居民档案（录入指标）、签约与随访（查看异常预警）

---

### 6.2 创新点② 随访智能闭环

**背景与价值**

随访是社区卫生中心的法定公卫工作，但传统流程依赖医护人工提醒，容易遗漏。本系统将随访任务的生成、推送、执行、记录全流程打通，实现真正的业务闭环。

**实现方向**

- **触发时机**：医生完成接诊后，系统检测该居民是否为慢病患者（根据历史诊断记录的病种标签判断）
- **模板匹配**：若识别为慢病患者，系统根据病种从预置的随访模板库中匹配对应模板，模板内容包含建议随访时间（如 1 个月后）和随访重点问题清单
- **医生确认**：接诊完成后弹出随访计划确认步骤，医生可调整时间和内容后提交
- **居民通知**：计划确认后，居民端就诊记录对应条目下自动出现随访提醒，注明随访日期和主要内容
- **护士执行**：到达随访日期时，签约与随访模块中该任务变为待执行状态，由护士进行电话联系或上门随访，填写随访结果（居民状态、指标反馈、是否需要复诊）
- **闭环完成**：护士提交结果后任务状态变为已完成，就诊记录和随访记录均归档保存

**涉及模块**

- 医护端：接诊工作台（触发生成计划）、签约与随访（管理与执行任务）
- 居民端：就诊记录（接收随访提醒）

---

### 6.3 创新点③ 家庭代管的分级数据权限

**背景与价值**

社区卫生中心的主要服务群体包含大量老年人（不会使用手机）和儿童（无行为能力），子女或监护人代操作是真实的高频需求。但简单地共享账号会带来隐私泄露风险，需要细粒度的数据权限控制。

**实现方向**

- **添加成员**：居民在家庭成员模块填写成员信息，系统创建代管关系记录，记录主账号、成员、关系类型和授权状态
- **授权范围**：代管人可操作的范围为代预约挂号、代查就诊记录、代查疫苗接种记录，不包含查看健康档案中的诊断详情
- **敏感信息隔离**：就诊记录支持标记为敏感（如特定科室的就诊），代管查询接口中敏感记录不返回，居民本人始终可见
- **权限校验**：后端每次代管操作都校验当前登录用户是否持有该成员的有效授权，防止越权访问
- **撤销授权**：被代管人可随时撤销授权，撤销后代管人立即失去对应数据的访问权限

**技术方向**：使用 Spring Security 结合自定义权限校验逻辑实现数据行级权限控制，三端共用同一套 JWT 认证体系。

**涉及模块**

- 居民端：家庭成员（管理代管关系）、预约挂号（代预约逻辑）、就诊记录（代查时过滤敏感数据）

## 7. 核心业务流程

### 7.1 主线诊疗闭环（10 步）

| 步骤 | 操作主体 | 所在模块 | 说明 |
|---|---|---|---|
| 1 | 管理员 | 排班管理 / 科室管理 | 配置科室信息、为医生护士排班、设置号源数量 |
| 2 | 居民 | 预约挂号 | 选择科室和医生，预约特定时间段；已签约医生优先展示 |
| 3 | 护士 | 接诊工作台 | 居民到诊后，护士登记到诊状态，呼叫排队叫号 |
| 4 | 医生 | 接诊工作台 | 接诊居民，填写诊断结论，开具处方；就诊记录自动保存 |
| 5 | 系统 | 自动触发 | 检测是否为慢病患者，若是则提示医生生成随访计划 |
| 6 | 医生 | 接诊工作台 | 确认或修改系统生成的随访计划后提交 |
| 7 | 居民 | 就诊记录 | 在就诊记录中查看本次诊断、处方及随访提醒 |
| 8 | 护士 | 签约与随访 | 随访到期时，护士执行电话或上门随访，填写随访结果 |
| 9 | 医生 | 签约与随访 | 查看随访结果，若指标持续异常则安排下次就诊或调整方案 |
| 10 | 居民 | 我的档案 | 在健康档案中查看历次慢病指标趋势图，了解健康变化 |

### 7.2 签约辅助流程（4 步）

| 步骤 | 操作主体 | 所在模块 | 说明 |
|---|---|---|---|
| A1 | 居民 | 我的档案 → 签约入口 | 居民发起家庭医生签约申请 |
| A2 | 管理员 | 签约管理 | 审核签约申请，确认医生负载合理后批准或驳回 |
| A3 | 医生 | 签约与随访 | 签约确立后，该居民出现在医生的签约居民列表中 |
| A4 | 居民 | 预约挂号 | 下次预约时，签约医生优先展示在推荐位置 |
