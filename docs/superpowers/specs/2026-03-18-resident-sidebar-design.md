# 居民端 PC 侧边栏布局改造设计

## 背景

居民端共有 **11 个功能模块**，但当前 `ResidentLayout.vue` 仅在导航中暴露 5 个（首页/预约/档案/消息/我的），其余 6 个模块（就诊记录、疫苗接种、候诊进度、家庭成员、健康资讯、我的随访）仅能通过首页快捷入口进入——PC 桌面端用户无法从导航直达这些页面。

同时，管理端和医护端已使用成熟的 `AdminLayout.vue`（左侧边栏 + 顶部栏 + 全宽内容区），信息密度和空间利用率远优于居民端。

## 模块清单与导航分组

将 11 个模块按使用场景分为 **3 组**：

| 组名 | 模块 | 路由 |
|------|------|------|
| **核心服务** | 服务首页 | `/resident/home` |
| | 预约挂号 | `/resident/appointment` |
| | 候诊进度 | `/resident/queue-progress` |
| | 就诊记录 | `/resident/visit-records` |
| **健康管理** | 健康档案 | `/resident/health-record` |
|  | 疫苗接种 | `/resident/vaccine` |
| | 我的随访 | `/resident/follow-up` |
| | 健康资讯 | `/resident/article` |
| **个人** | 家庭成员 | `/resident/family` |
| | 消息通知 | `/resident/message` |
| | 个人中心 | `/resident/profile` |

## 方案对比

### 方案 A：复用 AdminLayout（共享组件）

直接让居民端也使用 `AdminLayout.vue`，像 admin/medical 一样通过 `computed` 切换导航数组。

- ✅ 零重复代码
- ❌ AdminLayout 的「管理后台」气质对居民不友好（暗黑模式开关、角色标签等与居民无关）
- ❌ 三端耦合在一个组件中维护困难

### 方案 B（推荐）：复刻 AdminLayout 但定制居民端视觉

新建一个独立的 `ResidentLayout.vue`（保持当前文件名），但内部结构改为 Admin 风格的侧边栏布局，同时保留居民端的视觉气质（亲和色调、无暗黑模式、无角色标签）。

- ✅ 侧边栏全导航暴露
- ✅ 独立维护，不影响管理端
- ✅ 可定制居民端特有元素
- ✅ 移动端底部 Tab 保留（与 Admin 一致的响应式策略）

### 方案 C：顶部导航 + 全宽

保留顶部导航，去掉 max-width 限制。

- ❌ 之前的分析已证实此方案不适合 11 个模块

## 推荐方案详细设计

### 布局结构

```
┌──────────────────────────────────────────────────────────┐
│ [el-aside 220px / 64px]  │  [el-header 顶栏]            │
│                           │  · 收缩按钮 · 页面标题        │
│  品牌：社区卫生服务中心      │  · 用户名 · 退出             │
│  ─────────────────────── │──────────────────────────────│
│                           │                              │
│  ▸ 核心服务               │  [el-main 全宽内容区]          │
│    服务首页               │                              │
│    预约挂号               │   <router-view />            │
│    候诊进度               │                              │
│    就诊记录               │                              │
│                           │                              │
│  ▸ 健康管理               │                              │
│    健康档案               │                              │
│    疫苗接种               │                              │
│    我的随访               │                              │
│    健康资讯               │                              │
│                           │                              │
│  ▸ 个人                   │                              │
```

### 视觉定制（与 AdminLayout 的差异）

| 特性 | AdminLayout | 居民端新 Layout |
|------|------------|--------------|
| 品牌 Logo | `HeartPulse` 图标 | `Plus` 十字图标（沿用当前） |
| 暗黑模式切换 | ✅ 有 | ❌ 不需要 |
| 角色标签 | `管理员` / `医护` | 不显示 |
| 侧边栏分组 | 无分组，平铺 | **有分组标题**（核心服务/健康管理/个人） |
| 导航图标风格 | 与管理端一致 | 使用更亲和的 Lucide 图标 |
| 移动端 | 底部 5 Tab + 更多抽屉 | 底部 5 Tab + 更多抽屉（同策略） |
| 配色 | `--primary` 绿色 | 同 `--primary`，统一 |

### 移动端策略

与 AdminLayout 完全一致：
- 桌面端（≥ 768px）：侧边栏 + 顶栏
- 移动端（< 768px）：隐藏侧边栏 → 底部 Tab（5 个主入口）+ "更多"抽屉

移动端 5 个主 Tab：首页、预约、档案、消息、我的（保持不变）

### 各页面改动

侧边栏改造完成后，各页面的 `max-width` 和居中限制全部移除：

| 页面 | 当前 max-width | 改造后 |
|------|--------------|--------|
| HomePage | 1100px | 全宽 |
| ProfilePage | 1000px | 全宽 |
| VaccinePage | 1100px | 全宽 |
| AppointmentPage | 1100px | 全宽 |
| MessagePage | 无限制 | 保持全宽 |
| VisitRecordPage | 无限制 | 保持全宽 |
